# TDD Workflow Quick Reference

每個開發循環的檢查清單，確保不跳步。

## 開始新功能前

- [ ] 確認有 plan.md 或明確的功能需求
- [ ] 確認目前所有測試都通過
- [ ] 確認沒有未 commit 的變更

## Red → Green → Refactor 循環

### Red 階段

- [ ] 寫一個且只寫一個失敗測試
- [ ] 測試名稱描述行為，不描述實作
- [ ] 跑測試，確認新測試失敗且失敗原因符合預期
- [ ] 確認其他既有測試仍然通過

### Green 階段

- [ ] 只寫讓測試通過的最少程式碼
- [ ] 不寫測試沒要求的功能
- [ ] 跑全部測試，確認全部通過

### Refactor 階段

- [ ] 檢視是否有重複需要消除
- [ ] 檢視命名是否清晰表達意圖
- [ ] 每個重構步驟後跑測試
- [ ] 重構完成後全部測試仍通過

## Commit 檢查

- [ ] 所有測試通過
- [ ] 所有 compiler/linter 警告已解決
- [ ] 變更是單一邏輯單位
- [ ] Commit message 標明是 structural 或 behavioral
- [ ] 結構性與行為性變更沒有混在同一個 commit

## 修 Bug 流程

1. [ ] 寫 API 層級失敗測試（從使用者角度）
2. [ ] 寫最小失敗測試（精確定位問題）
3. [ ] 修復讓兩個測試都通過
4. [ ] 考慮是否需要重構防止再發
