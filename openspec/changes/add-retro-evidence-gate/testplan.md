## Missing Coverage

以下為契約層 [doc] TC 之外、pytest 不可驗（[manual]）或需 store 端支援才可 [mech] 的殘餘，明列不假裝已覆蓋：

- `retro-evidence-gate#unclassified-blocked-from-promotion` — [doc] 只驗 runbook 明文有 gate ordering；**LLM 執行期是否真的不跳過 gate 屬 [manual]**，建議增 REG-DT-001m：真實 pr-retro session 餵未分級 item、人工觀察是否被擋。
- `retro-evidence-gate#probe-invalid-demoted-not-dropped` — 「修一次仍失敗」的重試次數邊界（0/1/2 次）未 BVA，建議增 [manual] BVA 於重試觸發點。
- `retro-evidence-gate#tier3-parked-not-always-loaded` — typed-lessons 寫入 `conf≤4, parked` 屬 store 行為；若 typed-lessons 有可呼叫 API，補 [mech] 斷言寫入 record 的 `conf`/`parked` 欄位。
- `retro-evidence-gate#recurrence-unparks-still-needs-evidence` — recurrence 計數 1→2 的 store 端遞增未 [mech] 驗；若計數邏輯落在可測純函式內，補 [mech]。
- （AC-003 修復指示）`check_rule_evidence` 回傳的違規訊息**本身**是否含修復指引未驗，建議 REG-VL-002 [mech]：斷言違規清單每項含「如何補證據標記」文字。
- （合法錨點正例）合法證據標記（`<!-- verified: probe -->` / `(Source: PR #NNN` / `Probed.`）只有 fail 案例、缺 pass 案例，建議 REG-EG-002 [mech] EP：新 rule 檔**附**各合法錨點 → 函式回空清單、hook exit 0。
