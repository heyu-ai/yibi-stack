## 可測性前提（讀 TC 表之前必讀）

pytest 唯一能斷言的表面是 **SKILL.md 這份文件**，以及
`plugins/pr-flow/skills/pr-cycle-deep/scripts/tests/test_convergence_contract.py` 這個機械檢查器。
所有 AC 中關於 **aggregator / lead 行為** 的部分都是 LLM 執行期行為，**pytest 無法驗證**。
因此每個 TC 的 Test Purpose 都帶前綴，讀者不得把 doc-contract 的 PASS 誤讀為行為證明：

| 前綴 | 意義 | 證明了什麼 | 沒證明什麼 |
|------|------|-----------|-----------|
| `[mech]` | 對純函式餵合成 fixture | 檢查器本身正確且會對壞輸入變紅 | — |
| `[doc]` | 對真實 SKILL.md 斷言 | runbook **有明文寫**這條規則 | lead 會不會遵守 |
| `[manual]` | 人工餵合成 review payload 觀察輸出 | 某一次執行的行為符合 | 任意未來 PR 的行為 |

### 實作前提：契約模組必須暴露純函式

`test_convergence_contract.py` 對應的檢查邏輯 **MUST** 以純函式形式暴露：

```python
def check_convergence_contract(text: str) -> list[str]:
    """回傳失敗訊息清單；空清單代表全數通過。"""
```

理由：**只對真實檔案斷言的測試檔，無法測試自己的失敗路徑**。沒有純函式入口，
PRC-DT-002 / PRC-EG-001 / PRC-EG-006 等「錨點消失時必須變紅」的負向案例全部無法實作，
檢查器就退化成「永遠綠、無資訊量」的裝飾。此前提屬 Step 3 設計決策，不是測試細節。

### Unicode 讀取要求（host rule 13）

所有錨點比對 **MUST** 以 `Path.read_text(encoding="utf-8")` 讀原始 UTF-8。
錨點含 `（`、`≤`、`——` 等全形／CJK 字元；用 shell `echo` 轉換或 ASCII 替代會**靜默不匹配**、
不報錯，導致整批檢查 no-op。

  行動：補一個錨點，斷言 runbook 要求在 R1 收尾時**記錄** SHA；沒有它，審查面規則不可實作。

  行動：補一個 TC 斷言 runbook 要求對 baseline SHA 做新鮮度檢查。目前不存在。
